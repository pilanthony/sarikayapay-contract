# SarikayaPay

## Project Description

### 🧩 Problem

Micro, small, and medium enterprises (MSMEs) and local merchants in the Philippines suffer from high transaction settlement fees, slow payout times, and a lack of open, immutable ledger infrastructure. Traditional financial reporting tools are centralized, expensive, and out of reach for the unbanked vendor ecosystems.

## Project Vision

### 🌟 Vision

To build a zero-barrier decentralized accounting environment across the Philippines, enabling micro-merchants to maintain secure, tamper-proof business financial histories on a global network.

## 🎯 Purpose

Our team built SarikayaPay to offer small business ecosystems an immediate, automated transaction logging framework that bypasses traditional banking costs and provides true ledger autonomy.

## 👥 Target Users

- **User Persona 1:** Local Sari-Sari Store Owners and market micro-merchants tracking daily settlement tallies.
- **User Persona 2:** Unbanked Filipino vendors and independent service providers requiring verified payment ledgers.

## Key Features

### ✨ Features

- **Secure Ledger Logs:** Instantly records structured payment details onto the blockchain.
- **Gas-Optimized Profile:** Built entirely using low-overhead storage patterns to minimize network fees.
- **Strict Parameter Boundary:** Programmed validation loops to safely reject malformed financial entries.

## 🛠️ Tech Stack

- **Frontend:** React, TypeScript, Vite
- **Backend:** Rust, Cargo
- **Blockchain:** Stellar (Soroban / Horizon API / Stellar SDK)
- **Other tools:** VS Code, GitHub Desktop

## 🚀 How to Run Locally
